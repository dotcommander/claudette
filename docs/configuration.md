# Configuration

Claudette's configuration lives in two places: a JSON file for source directories and an environment variable for output format.

## Config File

```bash
cat ~/.config/claudette/config.json
```

```json
{
  "source_dirs": [
    "/Users/you/.claude/kb",
    "/Users/you/.claude/skills",
    "/Users/you/.claude/agents",
    "/Users/you/.claude/commands"
  ]
}
```

`claudette install` writes this file on first run. Edit it directly — claudette re-reads it on every hook invocation, so changes take effect immediately without restarting anything.

## Source Directories

`source_dirs` tells claudette where to find `.md` files to index. Entries are deduplicated and non-existent paths are silently skipped.

### Default Directories

These four directories are always scanned, whether or not they appear in `source_dirs`:

| Directory | Entry type |
|-----------|-----------|
| `~/.claude/kb/` | `kb` |
| `~/.claude/skills/` | `skill` |
| `~/.claude/agents/` | `agent` |
| `~/.claude/commands/` | `command` |

Directories listed in `source_dirs` are processed first, then defaults are appended. This means custom directories get priority in the source order but doesn't affect scoring — scoring is query-driven, not position-driven.

### Adding Custom Directories

Append any path to `source_dirs`:

```json
{
  "source_dirs": [
    "/Users/you/.claude/kb",
    "/Users/you/.claude/skills",
    "/Users/you/.claude/agents",
    "/Users/you/.claude/commands",
    "/Users/you/work/team-kb",
    "/Users/you/projects/myapp/.claude/kb"
  ]
}
```

Then rebuild the index:

```bash
claudette scan
```

Useful additions:

- **Team knowledge base** — a shared repo of KB articles checked out locally
- **Project-local knowledge** — `.claude/kb/` alongside your project's code
- **Vendor or framework notes** — a directory of `.md` files about third-party libraries your team maintains

### Plugin Directories

Claudette automatically includes components from any Claude Code plugin installed via `~/.claude/plugins/`. Plugin install paths are read from `~/.claude/plugins/installed_plugins.json` at runtime. You don't configure this manually — it happens automatically whenever a plugin is installed or removed.

## Environment Variables

| Variable | Values | Default | Description |
|----------|--------|---------|-------------|
| `CLAUDETTE_OUTPUT` | `full`, `compact` | `full` | Controls the format of surfaced entries in hook output |

### Full output (default)

Shows the file path relative to `~/.claude/` and the entry's H1 heading:

```
  kb/go/concurrent-map-race.md — Concurrent Map Read/Write Race Condition [matched: goroutine, race]
  kb/go/sqlite-driver-name.md — SQLite Driver Name is sqlite Not sqlite3 [matched: sqlite]
```

Use full mode when you want Claude to know exactly where each entry lives so it can read the file if needed.

### Compact output

Shows the frontmatter `name` and `description` instead of the file path:

```
  concurrent-map-race — Races on concurrent map read/write [matched: goroutine, race]
  sqlite-driver-name — Driver name is sqlite not sqlite3 [matched: sqlite]
```

Use compact mode when entries have long paths that add noise, or when descriptions are more informative than filenames.

Set `CLAUDETTE_OUTPUT` in your shell profile:

```bash
export CLAUDETTE_OUTPUT=compact
```

Or add it to Claude Code's environment in `~/.claude/settings.json`:

```json
{
  "env": {
    "CLAUDETTE_OUTPUT": "compact"
  }
}
```

> **Tip:** Compact mode requires good `description` frontmatter. If your entries have no `description`, compact output shows only the name — less useful than full mode.

## Config Directory Files

Claudette stores three files under `~/.config/claudette/`:

| File | Contents | Edit? |
|------|---------|-------|
| `config.json` | Source directory list | Yes — edit freely |
| `index.json` | Cached entry index with BM25 weights | No — regenerated by `scan` |
| `usage.log` | Append-only hit log | Truncate if large |

Don't edit `index.json` — any manual edit is overwritten the next time claudette detects a stale index. `usage.log` is append-only; truncate it when it grows large:

```bash
> ~/.config/claudette/usage.log
```
